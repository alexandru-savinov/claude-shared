Implement v0 of the deliberative Council — three independent assessors (opportunity, risk, compliance-security) that take a proposed consequential action and emit a structured verdict + a logged record. It proposes and gates; it never acts. The human is the apex.

## Context

Design source (read first): `~/.claude/index/governance-sancta-2026-06-20.md` (the council section) and `~/.claude/index/dreams/2026-06-20.md`. Grounded in the user's `AGI/ASI Interaction Checklist` (github.com/alexandru-savinov/Sancta — §3 guardrails, §4 forecast+self-critique, §6 tripwires, §7 review, §8 stops) and Kuzea's tiered-autonomy / immutable-auditor model (`~/nixos-config/docs/kuzea-plan.md`).

Build as a Claude Code skill **in this repo (claude-shared)** under `content/skills/council/`. The skill is invoked with a proposed action; it runs the three assessors INDEPENDENTLY, synthesizes a decision, logs it, and surfaces the verdict.

Core logic (from the design):
- **OPPORTUNITY** (advocate): is it worth it? value / upside / rationale.
- **RISK** (skeptic): what breaks? blast radius · reversibility · worst case · top failure modes — adversarial, tries to refute.
- **COMPLIANCE & SECURITY** (the law): is it ALLOWED per the charter? **HARD VETO** on violation.
- Powers differ: compliance = hard veto; risk = a WEIGHT that sets the tier; opportunity = proposal only.
- Decision = **block** (if compliance veto) → else **escalate-to-human** (if risk ≥ medium, OR out-of-bounds/novel, OR it touches money/software/matter/secrets/network-exposure/irreversible/outward) → else **proceed**.
- The human is the apex; high-risk / edge always escalates, never auto-proceeds.

v0 is a CC skill only. Fleet/commons integration, non-CC agents, auto-invocation, and signed verdicts are **v1 — out of scope here.**

### Task 2: The three independent assessor templates
- [ ] `content/skills/council/assessors/opportunity.md` — an advocate prompt: given a proposal, output ONLY the opportunity section (value, rationale). Must not reference or defer to the other assessors.
- [ ] `content/skills/council/assessors/risk.md` — an adversarial skeptic prompt: try to REFUTE the proposal; output the risk section (tier, blast radius, reversibility, top failure modes). Default toward caution under uncertainty. Independent framing.
- [ ] `content/skills/council/assessors/compliance.md` — a rules-checker prompt: check the proposal ONLY against CHARTER.md; output compliance (allowed / veto + reason) + any tripwires fired. Independent framing.
- [ ] Add a test that each template, run on the sample proposal, produces a valid section of the schema. Independence is a hard requirement — the three prompts must be self-contained and must NOT reference each other's outputs.

### Task 3: Synthesis + logging

## Constraints
- The Council PROPOSES and GATES — it must NEVER take a consequential action itself (no deploys, secrets, network exposure, irreversible or outward acts). It only reasons, logs, and surfaces a verdict.
- The three assessors MUST be independent: distinct, self-contained prompts, no shared framing; risk must be free to refute opportunity. Real adversariality — not a rubber stamp.
- Compliance has a HARD VETO; risk is a WEIGHT (sets tier/escalation); opportunity only proposes. The HUMAN is the apex — high-risk / edge / novel / human-gate-category ALWAYS escalates, never auto-proceeds.
- The CHARTER is immutable at runtime — the council cannot rewrite its own rules.
- v0 scope is a Claude Code skill in this repo ONLY. Do NOT build fleet/commons integration, non-CC agents, auto-invocation, or signed verdicts (that is v1). Do NOT touch the NixOS fleet config, secrets, prod hosts, or anything outside `content/skills/council/`.
- Data-only, local, reversible, additive. Pure Node for scripts (no npm deps). Each task includes its tests; the Task-5 runner is the closing check and must pass before done.
- Follow claude-shared conventions: a skill is `content/skills/<name>/SKILL.md` with `name` + `description` frontmatter.
